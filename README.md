# signal-mirror-tree
Deterministic reflection decision tree system
# Flow Explanation — Signal Mirror Tree

## Overview

This project is a rule-based system that helps users reflect on their day using fixed choices
At each step:

* the user selects one option
* the system records that choice
* the system moves to the next predefined step

There is **no guessing, no AI generation, and no randomness**.

---

## How the Flow Works

The reflection happens in three stages:

### 1. Control Signal (How you handled your day)

The user answers questions about:

* what happened
* how they reacted

The system checks:

* did the user take action? (internal)
* or feel controlled by events? (external)

---

### 2. Value Signal (What you gave)

The user reflects on:

* whether they contributed
* or focused on what they received

The system identifies:

* contribution mindset
* or expectation mindset

---

### 3. Impact Signal (Who you focused on)

The user considers:

* themselves
* their team
* or others

The system tracks:

* self-focused vs others-focused thinking

---

## How Decisions Are Made

Every question has **fixed options**.

Each option:

* adds a signal (like internal / external)
* leads to a specific next step

Example:

* “My choices” → internal
* “Waiting on others” → external

No interpretation is needed.

---

## Final Summary

At the end:

* the system counts signals
* finds which side is stronger for each axis

Output looks like:

* Control → internal
* Value → contribution
* Impact → other

---

## Why This Is Reliable

* same answers → same result
* no AI guessing
* no unclear inputs
* every path is predefined

---

## Key Idea

The system does not judge or predict.

It simply:

* records choices
* organizes them
* reflects them back clearly
