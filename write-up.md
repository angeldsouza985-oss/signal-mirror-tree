# Write-up — Signal Mirror Tree

## 1. Why I Chose These Questions

I designed the questions to surface behavior across three key dimensions: control, value, and impact.

For the first axis (control), the goal was to understand whether the user sees themselves as influencing outcomes or being influenced by external factors. So the questions focus on reactions and decision-making during the day.

For the second axis (value), I wanted to capture whether the user is oriented toward contribution or expectation. The questions highlight whether the user focused on helping, improving, or instead felt a lack of recognition or fairness.

For the third axis (impact), the aim was to expand perspective. The questions move from self-focused thinking to awareness of team, colleagues, or end users.

All questions use fixed options to ensure clarity and remove ambiguity.

---

## 2. How I Designed the Branching

The branching follows a layered structure:

1. Control → establishes baseline mindset
2. Value → evaluates behavior
3. Impact → expands perspective

Each axis uses:

* a primary classification question
* a deeper follow-up question
* a reflection node

Decision nodes route users based on their selected options, not interpretation.

### Trade-offs:

* I chose **clarity over flexibility**
  → fixed options instead of free text

* I avoided dynamic responses
  → all reflections are predefined

* I limited depth per axis
  → to keep the flow simple and readable

This ensures the system remains deterministic and easy to trace.

---

## 3. Psychological Sources

The design is inspired by:

* **Locus of Control (Julian Rotter)**
  → used in the Control axis

* **Growth Mindset (Carol Dweck)**
  → reflected in how users interpret outcomes

* **Psychological Entitlement vs Contribution**
  → used in the Value axis

* **Organizational Citizenship Behavior**
  → capturing voluntary contribution

* **Self-Transcendence (Maslow)**
  → used in the Impact axis

* **Perspective-Taking (Batson)**
  → expanding awareness beyond self

These concepts guided how questions were framed and how responses were categorized.

---

## 4. What I Would Improve

With more time, I would:

* Increase the number of questions per axis to improve accuracy
* Add more nuanced signal tracking (e.g., mixed states)
* Create multiple summary templates based on different signal combinations
* Add visualization of user patterns over time
* Improve the depth of reflections while still keeping them deterministic

---

## 5. Final Note

The system is intentionally designed to be simple, deterministic, and fully traceable.

Every path is predefined, every output is fixed, and no AI-based interpretation is used.

The goal is not to predict or judge behavior, but to reflect it clearly and consistently.
